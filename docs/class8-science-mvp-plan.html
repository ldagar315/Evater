01 / DECISION

Choose Science first.

Science is the better first subject for a fast, deterministic MVP. Concepts, experiments, classifications, cause-and-effect relationships, and data interpretation map naturally to reviewable MCQs.

Working assumption: NCERT/CBSE, Class 8, English, current Science curriculum version. The new “Curiosity” material means we must version the curriculum and avoid mixing old and new content. Official bridge programme ↗
Free lanePublished MCQs, deterministic scoring, explanations, mastery, and recommendations.
Paid laneNovel tests, handwritten OCR, subjective evaluation, personalized feedback, and viva.
Pilot sizeOne chapter first. Publish 100 approved questions before scaling to the subject.
Question typeSingle-answer, four-option MCQs only for the MVP. Add other types later.
02 / CONTENT

Build a bank that can route.

The target is 100 published questions per chapter, with enough structured metadata to move a student toward foundation, practice, review, or challenge questions.

Difficulty40 easy / 40 medium / 20 hardControls progression and challenge.

Concept coverageEvery major concept represented; no concept owns more than 20% of the bank.Supports concept-level mastery.
Question styleMostly text, with scenario, experiment, table, data, and diagram variants.Creates meaningful practice variety.
Quality gateExactly one correct answer, four unique options, explanation, source reference, reviewer approval.Prevents ambiguous or unsafe publishing.

04 / CONTRACTS

Small APIs, clear boundaries.

The free bank path is versioned and does not call an LLM. Existing AI endpoints stay available behind paid entitlement checks while the frontend migrates.

Example create-test request

POST /api/v1/tests

{
  "chapter_id": "uuid",
  "mode": "practice",
  "question_count": 10
}

The response returns five questions first. The next five are selected after the first block, making mid-test routing possible without sending the entire bank to the client.

05 / PERSONALIZATION

Adaptive, but explainable.

The MVP should use deterministic heuristics rather than a complicated recommendation model. That keeps the behavior testable and easy to tune.

Below 40%Route to easy questions, weak concepts, and prerequisite concepts.
40–75%Use a mixed block at the current difficulty band.
Above 75%Promote toward application and harder questions.
Repeat controlExclude the last 20 chapter questions and apply spaced review later.

Mastery update

new_mastery = 0.70 * previous_mastery
            + 0.30 * current_block_accuracy

Difficulty bands are easy below 45 mastery, medium from 45–74, and hard from 75 onward. Question priority is approximately 55% weak concepts, 20% review-due concepts, 15% unseen questions, and 10% challenge/exploration.

Product language: show “needs review”, “on track”, or “challenge next”. Do not claim that the MVP has measured a student’s exact ability.
06 / PIPELINE

Curated ingestion first.

For speed and safety, start with a curated source registry and adapters. Do not build a general internet crawler.

  1. Register source — URL, publisher, license, curriculum version, and attribution.
  2. Fetch source — rate limits, content hash, retry policy, and source status.
  3. Extract text — HTML/PDF parser; scanned material requires OCR and review.
  4. Normalize — chapters, concepts, learning outcomes, and source locators.
  5. Generate candidates — manual or permitted offline batch generation.
  6. Validate — schema, exactly-one-answer, duplicate, length, and scope checks.
  7. Review — approve, reject, or request edits.
  8. Publish — only approved questions enter the student-facing bank.
  9. Refresh — detect source changes and create a new content version.
Rights boundary: use official metadata, openly licensed content, and original questions. Do not republish NCERT textbook passages, images, or copied solution-site material without the required permission. NCERT copyright notice ↗

08 / DELIVERY

Fast path to a real MVP.

  1. Freeze scope: Class 8, Science, English, one current curriculum version, one chapter.
  2. Build schema: migrations, RLS, question validation model, and seed curriculum rows.
  3. Build content loader: curated source adapter plus JSON/CSV import for fast review.
  4. Publish the bank: 100 approved, tagged questions with explanations.
  5. Build free APIs: test creation, five-question blocks, deterministic scoring, and results.
  6. Add routing: mastery updates, weak-concept targeting, difficulty promotion, and repeat avoidance.
  7. Wire the frontend: chapter → test → block → result → recommendation.
  8. Protect paid AI: entitlement checks around existing generation, answer evaluation, and viva flows.
